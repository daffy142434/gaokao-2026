# 对话记录 - 2026-06-25 (续)

## 用户提问
1. 设置省份和类型时没生效
2. 选择了省份之后，会话还是在说没有填写省份
3. 首页的问题改为引导型的（3个）
4. 填写的分数/省份信息，应该是和会话关联的
5. 新建会话时，没有弹窗填写信息了

## 执行结果

### 1. Profile改为per-session存储
- 移除全局 `LS_PROFILE` 常量
- profile数据存入主存储结构 `LS_KEY` 的 `profiles` 字段，按sessionId索引
- `loadSessionProfile(sid)` 切换会话时自动加载对应profile
- `saveProfile()` 保存时将profile关联到当前会话
- `newChat()` 重置profile为空，开启全新会话
- `loadSession()` 切换会话时同步加载该会话的profile

### 2. AI上下文修复
- 客户端 `sendMessage()` 请求体增加 `profile` 字段，携带当前省份/分数/类型
- 服务端 `/api/gaokao/chat` 接收 `profile` 参数，优先使用客户端传来的profile信息构建系统提示词
- 客户端传来的profile优先级高于从消息文本中正则提取的信息

### 3. 首页示例改为引导型问题（3个）
- "我想学计算机，600分左右能去哪些学校？"
- "想学医的话，5+3和八年制怎么选？"
- "留在省内还是出省？对本省院校比较纠结"

### 4. 选择器验证
- 头部省份/类型弹窗按钮和选择逻辑经检查无问题，选择后正确关闭弹窗并保存
- `selectProvince(p)` → 设置profile.province → `onProfileChange()` 保存并关闭弹窗
- `selectType(t)` → 设置profile.type → `onProfileChange()` 保存并关闭弹窗

### 5. 新建会话引导弹窗验证
- `newChat()` 清空currentSessionId、messages、profile后调用 `checkShowOnboarding()`
- 条件判断 `!currentSessionId && messages.length === 0` 满足时显示引导
- 删除会话时调用 `newChat(true)` 跳过引导直接回到首页
